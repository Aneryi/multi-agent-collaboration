# 项目需求文档：多智能体协作叙事引擎（场景驱动版）

**版本**：1.1  
**日期**：2026年6月1日  
**状态**：已优化，作为 MVP 开发基线  

## 1. 项目概述

### 1.1 项目名称

多智能体协作叙事引擎（Multi-Agent Storytelling Engine, MASE）

### 1.2 项目目标

构建一个基于 LangGraph 的多智能体叙事系统。用户创建故事会话、角色与场景后，系统按“角色生成意图 -> DM 裁决 -> 状态更新 -> 记忆沉淀”的流程自动推进叙事。用户仍保留导演权，可以控制场景起止、注入事件、编辑角色与导出故事。

### 1.3 MVP 范围

第一版聚焦后端核心能力和可验证的命令行/API 流程：

- 支持创建故事会话、角色、场景。
- 支持按轮运行当前场景，并返回每轮角色意图、DM 叙事和状态变化。
- 支持短期记忆、长期记忆摘要、关系分值和世界状态的 JSON 持久化。
- 支持本地规则型 LLM 适配器，保证无外部 API Key 时也能运行和测试。
- 预留真实 LLM 适配器接口，后续可接入 OpenAI、Anthropic 或本地模型。

暂不在 MVP 中实现完整前端、Redis/PostgreSQL、向量数据库、多人协作、配图、TTS 与完整内容审核服务。这些能力纳入后续阶段。

## 4. 非功能需求

| 需求ID | 需求描述 | 验收标准 |
| --- | --- | --- |
| N-01 | 单轮叙事生成时间应小于等于 10 秒（3-5 个角色，外部模型正常响应）。 | 95% 轮次满足。 |

## 5. 技术实现方案

### 5.1 总体架构

MVP 采用单体后端结构，内部按模块分层：

```text
[FastAPI/CLI]
    -> [StoryService]
    -> [LangGraph 编排图]
       -> generate_intentions
       -> adjudicate_scene
       -> apply_state_updates
       -> reflect_memories
    -> [JSONRepository]
    -> [LLMAdapter]
```

后续阶段再替换为 Redis、PostgreSQL、任务队列和前端应用。

### 5.2 核心技术栈

| 组件 | MVP 技术 | 后续扩展 |
| --- | --- | --- |
| 运行时 | Python 3.11 | Python 3.12 |
| API | FastAPI | SSE/WebSocket |
| 智能体编排 | LangGraph | 分布式 worker |
| 数据模型 | Pydantic | SQLModel/SQLAlchemy |
| 持久化 | JSON 文件 | PostgreSQL + Redis |
| LLM 调用 | 本地规则适配器 + LangChain 接口 | OpenAI/Claude/本地模型 |
| 测试 | pytest | 压测与评估集 |

### 5.3 场景运行流程

1. 读取会话、当前场景、激活角色和世界状态。
2. 角色节点并行或顺序生成结构化意图：
   - `character_id`
   - `action`
   - `dialogue`
   - `target_character_id`
   - `emotion`
   - `interrupt`
   - `impulse`
3. DM 节点接收意图列表，输出：
   - `narrative`
   - `world_delta`
   - `relationship_delta`
   - `memory_hints`
   - `debug_reason`
4. 状态更新节点追加叙事日志、意图日志和短期记忆。
5. 反思节点每 5 轮或场景结束时沉淀长期记忆。
6. 达到轮数上限或用户停止后保存状态。

## 6. API 设计

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/sessions` | 创建故事会话 |
| GET | `/api/sessions/{session_id}` | 读取会话详情 |
| POST | `/api/sessions/{session_id}/characters` | 创建角色 |

| GET | `/api/sessions/{session_id}/export` | 导出完整故事 Markdown |

## 7. 验收标准

| 测试场景 | 预期结果 |
| --- | --- |
| 2 个角色运行 3 轮 | 每轮包含两个角色意图和一段 DM 叙事。 |

## 8. 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
| --- | --- | --- | --- |
| LLM 输出格式不稳定 | 中 | 高 | 使用结构化输出 schema，失败时重试或降级规则解析。 |
| 成本随角色数增长过快 | 高 | 中 | 角色意图使用轻量模型，限制轮数和上下文长度。 |
| 记忆摘要丢失关键事实 | 中 | 高 | 短期记忆保留窗口，长期记忆定期反思，后续引入向量检索。 |
| 故事陷入重复 | 中 | 中 | DM 节点检测重复短语，注入环境变化或目标推进。 |
| 安全内容风险 | 中 | 高 | MVP 提示词约束和规则过滤，后续接入审核服务。 |

## 10. 术语表

- **DM**：Dungeon Master，叙事裁决与文本生成智能体。
- **意图**：角色在当前轮想执行的动作、对话、目标和情绪。
- **场景**：故事中的连续时空单元。
- **世界状态**：地点、道具、环境、全局事件标记等结构化状态。
- **记忆沉淀**：将短期事件摘要合并进角色长期记忆的过程。
